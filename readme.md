# Tender Analyzer Pipeline

Пайплайн для автоматической обработки тендеров: получение ссылок из почты, парсинг страниц, скачивание документации, извлечение текста, классификация и глубокий анализ с помощью RAG, формирование отчётов (HTML, Excel) с отправкой по email.

---

## 1. Требования к оборудованию (Hardware)

- **Процессор:** Intel Core i5 / AMD Ryzen 5 или аналогичный (рекомендуется 4+ ядра).  
- **Оперативная память:** Минимум 4-8 ГБ.  
- **Диск:** SSD, свободное место не менее 5-10 ГБ (зависит от количества скачиваемых файлов и кэшей эмбеддингов).  
- **Видеокарта (опционально):** Для ускорения эмбеддингов и работы LLM через Ollama используются **GPU ядра MPS (Metal Performance Shaders)** на Apple Silicon, а также дискретные GPU NVIDIA с 6+ ГБ видеопамяти. При отсутствии GPU всё будет работать на CPU, но медленнее.  
- **Сеть:** Стабильный интернет для загрузки страниц, файлов и доступа к почтовому серверу.

---

## 2. Требования к программному обеспечению (Software)

### Операционная система

- **Windows 10/11** (для обработки `.doc` требуется MS Word и библиотека `pywin32`).  
- **Linux (Ubuntu 20.04+)** – предпочтительна, так как проще установить все зависимости.  
- **macOS** – поддерживается, причём на Apple Silicon автоматически используются GPU ядра MPS для ускорения вычислений эмбеддингов.

### Python и окружение

- **Python 3.8 – 3.11** (3.12 может иметь проблемы с некоторыми библиотеками).  
- Виртуальное окружение (рекомендуется `venv` или `conda`).

### Сторонние утилиты и сервисы

| Компонент              | Требование                                                                                                 |
|------------------------|------------------------------------------------------------------------------------------------------------|
| **Ollama**             | Должен быть установлен и запущен. Модель (например, `qwen3:4b-instruct`) должна быть загружена.            |
| **Playwright**         | Устанавливается через `pip`, затем выполнить `playwright install chromium`.                                 |
| **Unrar**              | Для работы с RAR-архивами требуется утилита `unrar` в системе (Linux: `sudo apt install unrar`, Windows: установить вручную). |
| **MS Word**            | Только для обработки старых `.doc` файлов на Windows.                                                       |
| **IMAP / SMTP**        | Доступ к почтовому серверу (например, mail.ru, yandex.ru) с включённым доступом по паролю приложений.      |

### Python-зависимости

Все зависимости перечислены в `requirements.txt`. Основные пакеты:
- `aiohttp`, `aiofiles` – асинхронные загрузки  
- `playwright` – браузерная автоматизация  
- `sentence-transformers`, `chromadb`, `faiss-cpu` – RAG и эмбеддинги  
- `pandas`, `openpyxl` – работа с Excel  
- `pdfplumber`, `python-docx`, `beautifulsoup4` – извлечение текста  
- `rank-bm25`, `langchain`, `nltk` – вспомогательные инструменты  
- `rarfile`, `py7zr` – распаковка архивов  
- `python-dotenv` – загрузка переменных окружения

## 3. Подводные камни при развёртывании (Troubleshooting)

### 3.1. Авторизация на rostender.info

- В `config.AUTH_URL` должен быть указан правильный URL страницы авторизации.  
- Если после авторизации не находится элемент `.header-login__name`, проверьте, не изменилась ли структура сайта. Возможно, потребуется обновить селекторы в `file_parser.py`.  
- Иногда требуется ручная капча – в таком случае headless-режим может не сработать. Используйте `--headless False`, чтобы ввести капчу вручную.

### 3.2. Настройка почты (IMAP/SMTP)

- Для mail.ru и yandex.ru необходимо создать **пароль приложения** (не основной пароль).  
- В `config.py` проверьте, что `IMAP_SERVER` и `SMTP_SERVER` указаны верно.  
- Если письма не находятся – убедитесь, что папка "Входящие" не имеет других фильтров, а письма от отправителей из `ALLOWED_SENDERS` действительно приходят.

### 3.3. Проблемы с браузером Playwright

- Если Chromium не запускается в headless-режиме, попробуйте сначала запустить с графическим интерфейсом (`--headless False`) для отладки.  
- В Linux могут потребоваться дополнительные системные библиотеки (например, `libnss3`, `libatk-bridge2.0-0`). Playwright обычно устанавливает их автоматически, но иногда нужно доустановить вручную.  
- На Windows убедитесь, что антивирус не блокирует запуск браузера. На текущий момент не разобрался с headless браузером, у rostender.com хорошая защита не позволяющая авторизоваться в headless режиме.

### 3.4. Работа с архивами (RAR, 7z)

- Для RAR необходим **unrar** в системе.  
  - Windows: установите [WinRAR](https://www.win-rar.com/) или добавьте `unrar.exe` в PATH.  
  - Linux: `sudo apt install unrar`.  
  - macOS: `brew install unrar`.  
- Для 7z требуется установка `p7zip-full` (Linux) или добавление `7z.exe` в PATH (Windows).  
- Если архивы не распаковываются – проверьте, что библиотеки `rarfile` и `py7zr` установлены, а утилиты доступны.

### 3.5. Обработка .doc файлов (старый формат)

- На Windows для этого используется MS Word через COM-объекты (библиотека `pywin32`).  
- **На Linux и macOS обработка .doc невозможна** – такие файлы будут пропущены (если `PROCESS_DOC = False`). Рекомендуется заранее конвертировать .doc в .docx или PDF.  
- Если вы всё же включаете `PROCESS_DOC = True` на Windows, убедитесь, что MS Word установлен и лицензирован.

### 3.6. Ограничения LLM и производительность

- Модель Ollama (например, `qwen3:4b-instruct`) требует **не менее 4 ГБ оперативной памяти** для работы на CPU. При использовании GPU ядер MPS (Apple Silicon) или дискретной NVIDIA память видеокарты должна быть не менее 4–6 ГБ.  
- Если LLM отвечает слишком медленно или выдаёт пустые ответы, проверьте доступность Ollama (`ollama list`, `ollama ps`), а также настройки таймаутов в `ask_llama`.  
- Для больших тендеров (много файлов) процесс глубокого анализа может занимать часы. Увеличьте `MAX_CONCURRENT_TENDERS` осторожно, чтобы не перегрузить систему.

### 3.7. Кэширование и пути

- По умолчанию все данные сохраняются в папку `storage/`. Убедитесь, что у процесса есть права на запись в эту директорию.  
- Если вы используете относительные пути, оркестратор должен запускаться из **корневой папки проекта** (где лежит `main.py`).  
- При первом запуске будут созданы все необходимые подпапки, но если какие-то пути не могут быть созданы (например, из-за прав), программа упадёт.

### 3.8. Логирование

- По умолчанию логи пишутся в `storage/logs/orchestrator.log` с ротацией по размеру (10 МБ). Если диск заполнится, логи могут не писаться – следите за свободным местом.  
- При отладке полезно установить `LOG_LEVEL = "DEBUG"` в `.env` или в `config.py`.

### 3.9. Многопоточность и асинхронность

- Параллельная обработка тендеров (`MAX_CONCURRENT_TENDERS`) и скачивание файлов (`MAX_CONCURRENT_EXTRACT`) могут создавать высокую нагрузку. Настройте их в зависимости от вашего железа.  
- При использовании `asyncio` важно не смешивать блокирующие операции с асинхронными. В коде все блокирующие вызовы (например, работа с файлами) вынесены в потоки (`asyncio.to_thread`), но в редких случаях могут возникнуть дедлоки – проверяйте логи.

### 3.10. Зависимости от системных библиотек

- Для работы `pdfplumber` и `python-docx` дополнительных библиотек не требуется, но для `lxml` (используется в BeautifulSoup) может понадобиться компилятор C (на Linux).  
- На Alpine Linux могут потребоваться пакеты `gcc`, `musl-dev`, `python3-dev`.

## 4. Быстрая установка (кратко)

```bash
# Клонирование репозитория
git clone https://github.com/your-repo/tender-analyzer.git
cd tender-analyzer

# Виртуальное окружение
python -m venv venv
source venv/bin/activate   # Linux/macOS
# venv\Scripts\activate    # Windows

# Установка зависимостей
pip install -r requirements.txt

# Установка браузера Playwright
playwright install chromium

# Настройка переменных окружения
cp .env.example .env
# отредактируйте .env (указав AUTH_URL, MAIL, PASSWORD)

# Установка и запуск Ollama (пример)
curl -fsSL https://ollama.com/install.sh | sh   # Linux
ollama pull qwen3:4b-instruct

# Запуск пайплайна
python main.py --headless
